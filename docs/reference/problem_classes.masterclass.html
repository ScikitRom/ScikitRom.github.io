problem_classes.masterclass

Framework for static reduced-order modeling (ROM).

TL;DR

This module helps you:

  • Run offline full-model (FEM) solves to get data (“snapshots”)
  • Run online reduced-order (ROM) solves for fast prediction
  • Optionally use hyper-reduction (DEIM, ECSW, ECM) for quicker online solves, with error tracking

Main features:

  • A standard problem interface (base class) for consistency
  • A registry and factory to create various problem classes
  • Offline workflow for generating training data (snapshots) with the big/full model
  • Online workflow for using ROMs, with error and speed tracking
  • Ready-to-use hooks for hyper-reduction methods

Authors: Suparno Bhattacharyya, Ali Hamza Abidi Syed

Problem

problem_classes.masterclass.Problem()

Summary:

Defines what every parameterized FEM/ROM problem must have.
You make your own class from this, and supply the required methods, such as:

  • Mesh and basis setup
  • Bilinear and linear forms in affine terms (for easy ROM)
  • How parameters control coefficients in your problem
  • How to sample from your parameter space
  • How to solve full and reduced systems
  • Optional support for hyper-reduced solves (DEIM, ECSW, ECM)

Key methods you should provide (all listed below in the same format):

bilinear_forms
problem_classes.masterclass.Problem.bilinear_forms()

Returns all parts (“components”) of the bilinear form (stiffness, etc.) in affine form (for ROM).

domain
problem_classes.masterclass.Problem.domain()

Returns your mesh and FE bases.

fom_solver
problem_classes.masterclass.Problem.fom_solver()

Solves your big/full-order FEM system for given parameters.

hyper_rom_solver_deim
problem_classes.masterclass.Problem.hyper_rom_solver_deim()

Solves your reduced system using DEIM-based hyper-reduction.

hyper_rom_solver_ecm
problem_classes.masterclass.Problem.hyper_rom_solver_ecm()

Solves with ECM-based hyper-reduction.

hyper_rom_solver_ecsw
problem_classes.masterclass.Problem.hyper_rom_solver_ecsw()

Solves with ECSW-based hyper-reduction.

linear_forms
problem_classes.masterclass.Problem.linear_forms()

Returns all parts (“components”) of affine linear forms (force/load vectors etc.).

parameters
problem_classes.masterclass.Problem.parameters()

Produces a collection of parameter samples (for training/testing).

properties
problem_classes.masterclass.Problem.properties()

Returns a function that calculates coefficients depending on your current parameters.

rom_solver
problem_classes.masterclass.Problem.rom_solver()

Solves the reduced-order model.

fom_simulation

problem_classes.masterclass.fom_simulation(num_snapshots=32)

Offline workflow for collecting snapshot data:

  • Takes parameter samples
  • Solves the full (FEM) model for each sample
  • Saves the solutions and timings
  • Computes a mean/reference solution for centering
  • Stores all data for later ROM use

Attributes

Name Type Description
num_snapshots int How many samples to use
param_list array_like The sampled parameters
fos_solutions list of ndarray List of full-order FEM solutions
fos_time list of float Solve times for each snapshot
train_ref ndarray The average (“reference”) field computed
run_simulation
problem_classes.masterclass.fom_simulation.run_simulation()

Runs all full-order solves and saves each result:

  • Each snapshot checkpoint is written to .npz (safe/atomic)
  • If a checkpoint already exists, it skips and loads instead (resume on crash)
  • Uses hash for long filenames (handles Windows path limits)
  • If a checkpoint is corrupt, it recomputes and overwrites

rom_simulation

problem_classes.masterclass.rom_simulation(
    train_ref=None,
    test_ref=None,
    fos_solutions=None,
    train_mask=None,
    test_mask=None,
    V_sel=None,
    n_sel=None,
    N_rom_snap=None,
)

Online evaluation workflow for ROMs:

  • Loads data from disk
  • Picks test samples and (optional) reference full solutions
  • Runs your ROM and reconstructs the “full” fields if needed
  • Calculates error and speedup factors
  • Supports DEIM, ECSW, ECM solvers for hyper-reduced runs

Attributes

Name Type Description
V_sel ndarray Matrix (basis) used to reconstruct ROM solution
n_sel int Number of ROM modes used
param_list_test array_like List of test parameters
rom_error list of float % errors for each test case
speed_up list of float (FOM time)/(ROM time) for performance tracking

run_hyper_rom_simulation_deim
problem_classes.masterclass.rom_simulation.run_hyper_rom_simulation_deim(
    z,
    deim_mat,
    sampled_rows,
    full_order=True,
    new_params=None,
)

Runs the DEIM version of your ROM for all parameters:

  • For each parameter, runs the hyper-reduced solver
  • If needed, reconstructs the full solution for error comparison or just returns the reduced coordinates
  • Calculates relative error (%) and speed-up unless custom parameters (new_params) are provided
Parameters
Name Type Description Default
z array_like Weight vector from DEIM workflow required
deim_mat ndarray Interpolation matrix for DEIM required
sampled_rows array_like of int DOF row indices sampled by DEIM required
full_order bool If True, reconstruct full fields; if False, only reduced coords True
new_params sequence If provided, evaluates only for these params and skips error/speed metrics None
Returns
Name Type Description
hyper_rom_error list of float Errors (%) if new_params is not given
hyper_speed_up list of float Speed-up ratios if new_params is not given
hyper_rom_solutions list of ndarray List of reconstructed (or reduced) solutions (given when new_params is provided)
