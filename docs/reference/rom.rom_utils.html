rom.rom_utils

rom.rom_utils

Essential utilities for ROM (Reduced Order Model) workflows.

This module makes it easier and faster to:

  • Split your data into training and test sets for ROM studies
  • Generate parameter samples using the most popular methods
  • Update and improve your ROM basis (POD modes)
  • Reconstruct the full state from ROM (and add back the mean if needed)
  • Work with saving/loading ROM data
  • Solve nonlinear reduced and hyper-reduced problems using Newton’s method
  • Gather residuals (for hyperreduction, ECM, or DEIM training)

Authors: Suparno Bhattacharyya, Ali Hamza Abidi Syed

collect_residuals

rom.rom_utils.collect_residuals(
    NLS_train_ms,
    NLS_train_mean,
    V_sel,
    reconstruct_solution,
    Residual,
    training_params,
    assemble_kwargs,
    extra_kwargs=None,
    hyper_basis=None,
)

Purpose:
Evaluates the residual (model mismatch) for every training snapshot. Useful for DEIM- or ECSW-type hyperreduction training.

  • For each snapshot: projects to reduced coordinates, reconstructs the full field, computes residual, and stacks the result.

Computes and collects residual samples at the element–Gauss-point level (for ECM training).

compute_nonlinear_snapshots

rom.rom_utils.compute_nonlinear_snapshots(
    non_linear_func,
    fos_solutions,
    param_list,
)

Purpose:
Applies a nonlinear function to each full-order solution and its associated parameter, stacking the results for further POD/ROM analysis.

generate_gaussian_samples

rom.rom_utils.generate_gaussian_samples(dimensions, num_points, bounds)

Purpose:
Generates random samples from a normal distribution, centered and scaled within your provided bounds.

  • Mean is halfway between bounds, std is (upper-lower)/5.
  • No clipping: points can fall slightly outside the exact bounds.

generate_lhs

rom.rom_utils.generate_lhs(dimensions, num_points, bounds)

Purpose:
Generates Latin Hypercube samples (good for covering parameter space efficiently).

generate_sobol

rom.rom_utils.generate_sobol(dimensions, num_points, bounds)

Purpose:
Generates Sobol’ quasi-random samples, commonly used for uncertainty quantification and parameter studies.

  • num_points must be a power of two (e.g., 16, 32, 64).

latin_hypercube_train_test_split

rom.rom_utils.latin_hypercube_train_test_split(N_snap, train_percentage=0.8)

Purpose:
Splits indices via Latin hypercube for training and testing sets.
Gives boolean masks for both sets.

load_rom_data

rom.rom_utils.load_rom_data(self, rom_data_dir=None)

Purpose:
Loads ROM simulation data from a standard folder.

  • If self is None, just returns data.
  • If not, sets attributes on self.

reconstruct_solution

rom.rom_utils.reconstruct_solution(u_reduced, V_sel, mean)

Convert your ROM coordinates (usually very small arrays) back to the full field (e.g., displacement/temperature/etc.), re-adding the mean if appropriate.

rom_data_gen

rom.rom_utils.rom_data_gen(save_kw, problem_path)

Purpose:
Save ROM simulation results, including full-order solutions and all necessary metadata, to a standard folder for reproducibility.

  • Requires fos_solutions key in the data to save.

select_elements_and_gauss_weights

rom.rom_utils.select_elements_and_gauss_weights(

Purpose:
Turns flat lists of element/Gauss-point indices and weights into an easy-to-use dictionary: {element_index: [weight_per_gauss_point, …]}.
If a Gauss point in an element isn’t selected, its weight is 0.0.

sobol_train_test_split

rom.rom_utils.sobol_train_test_split(N_snap, train_percentage=0.8)

Purpose:
Same as Latin hypercube version, but for indices ordered by Sobol draw.

solve_linear

rom.rom_utils.solve_linear(
    A,
    b,
    *,
    backend='numpy',
    method='cg',
    rtol=1e-08,
    atol=0.0,
    maxit=500,
    petsc_pc='jacobi',
    petsc_options=None,
)

Purpose:
Solves linear system A x = b, letting you pick the solver backend (numpy, PETSc, etc.) and method (“cg”, “gmres”, etc.).

  • CG: for SPD matrices
  • MINRES: for general symmetric matrices
  • GMRES: for anything else

train_test_split

rom.rom_utils.train_test_split(N_snap, N_sel=None, train_percentage=0.8)

Purpose:
A simple way to make boolean masks for what’s train and what’s test in your set of indices (no fancy ranking).

  • Can pick only the first N_sel snapshots if you want to subselect.

update_basis

rom.rom_utils.update_basis(V, W_mu, max_modes=5)

Purpose:
Improve your current basis (V) by deflating out old directions, doing SVD on new data (W_mu), and adding up to max_modes new “directions” (modes).
Keeps your basis orthonormal by re-applying QR.

Summary:
This module ties together everything you need to efficiently run and manage data-driven reduced-order modeling with modern hyperreduction. Use for everything from sample design, through training, to nonlinear solving and simulation reproducibility!
